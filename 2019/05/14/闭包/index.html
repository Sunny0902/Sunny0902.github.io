2.js的内存管理

1.在JavaScript中有两种内存管理机制，分别是：

(1)标记清除法(大部分浏览器);

(2)引用计数法(低版本的IE浏览器).

2.1 引用计数

原理：每创建一个对象，都有一个数字用来标识当前对象被引用的次数。

2.2 内存泄露

1.内存泄露指的是应用程序的内存中出现了本应该释放但是却释放不了的对象

2.3 哪些情况会产生循环引用

a)闭包的循环使用

b)DOM与JavaScript的循环使用

2.4 怎么解决循环引用的问题

a)尽量在闭包中避免循环引用
b)避免闭包的使用

2.5 标记清除

基本思想：从某个跟节点出发，如果某个对象不可达，即说明是垃圾，就会被垃圾回收器清除内存。

4.JSON

1.JSON往往用于数据的载体，用户客户端和服务器通信的过程。
2.JSON是JS的原声格式，所以JSON和JS对象非常像，其实JSON也是一个JS对象.

3.JSON的注意点：

a)JSON对象的key必须是字符串,而且必须是双引号的字符串

b)JSON对象的value值可以是string Boolean number null object array，但是不可以是undefined

4.JSON对象和JSON字符串相互转换

a)将JSON对象转换为JSON字符串

b)将JSON字符串转换为JSON对象
